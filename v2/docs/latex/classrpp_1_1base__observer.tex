\subsubsection*{template$<$constraint\+::decayed\+\_\+type Type, constraint\+::observer\+\_\+strategy$<$ Type $>$ Strategy$>$\newline
class rpp\+::base\+\_\+observer$<$ Type, Strategy $>$}
Base class for any observer used in RPP. It handles core callbacks of observers. Objects of this class would be passed to subscribe of observable. 

\begin{DoxyWarning}{Warning}
By default \mbox{\hyperlink{classrpp_1_1base__observer}{base\+\_\+observer}} is not copyable, only movable. If you need to COPY your observer, you need to convert it to \mbox{\hyperlink{group__observers_gac1c33b452ca9e0a58fbde9da538ca5f1}{rpp\+::dynamic\+\_\+observer}} via \mbox{\hyperlink{classrpp_1_1base__observer_a601d0b432a3f88eeeee64e886efba9bd}{rpp\+::base\+\_\+observer\+::as\+\_\+dynamic}} 

Expected that observer would be subscribed only to ONE observable ever. It can keep internal state and track it it was disposed or not. So, subscribing same observer multiple time follows unspecified behavior. 

If you are passing disposable to ctor, then state of this disposable would be used used (if empty disposable or disposed -\/\texorpdfstring{$>$}{>} observer is disposed by default)
\end{DoxyWarning}
