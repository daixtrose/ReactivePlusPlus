Detailed Description

Disposable owns some resource and provides ability to dispose() it: destroy/remove/disconnect and etc.

In RPP it used as "reverse subscription": observable sets disposable to observer via set_upstream(disposable) with meaning "if you want to cancel me -> dispose this disposable"
